## Versioning Scheme

We use **semantic versioning** adapted for a living governance and philosophical framework:

### Format: `MAJOR.MINOR.PATCH[-Label]`

- **MAJOR** (e.g., 2.0.0): Significant architectural changes (e.g., full CUR implementation)
- **MINOR** (e.g., 1.6.0): New chapters, major feature additions, or substantial improvements
- **PATCH** (e.g., 1.5.1): Bug fixes, wording improvements, header updates, small refinements

### Status Labels

- **Official-Evergreen**: Stable, recommended version for general use
- **Draft**: Active development branch (unstable, may change frequently)
- **Legacy**: Older versions kept for historical reference
